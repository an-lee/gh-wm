# v2 breaking changes (planned / in progress)

This page consolidates **user-visible** cleanups targeted for a **v2** release. Until then, gh-wm remains backward compatible where noted; deprecations may emit warnings from [`internal/config/spec`](../../internal/config/spec/).

| Area | Change |
|------|--------|
| **`engine: copilot`** | Treated as **deprecated**: no stock CLI; use **`WM_AGENT_CMD`** or switch to **`claude`** / **`codex`**. v2 may remove the name unless a real implementation ships. |
| **`safe-outputs:` keys** | Prefer **dash** form (`add-labels`, `create-pull-request`) to match gh-aw. Underscore keys in YAML may warn; v2 may canonicalize [`output.json`](task-format.md) item types separately. |
| **`wm.state_labels`** | v2 may rename to a shorter block (e.g. `wm.states`) with typed `working` / `done` / `failed`. |
| **`tools:`** | v2 may require a structured shape (`allowed` / `denied`) and translate per engine (`--allowed-tools`, etc.). |
| **`timeout-minutes`** | Enforced inside **[`engine.RunTask`](../../internal/engine/runner.go)** so library callers and CLI share the same deadline (parent `context` deadlines still cap the run when stricter). |
| **Artifacts** | Each run writes **`run.json`** (merged **`meta.json`** + outcome fields) alongside **`meta.json`** and **`result.json`**. |
| **GitHub API** | Default **`internal/ghclient`** uses the **`gh` CLI** subprocess (`gh api`). Set **`GH_WM_REST=1`** to use **`go-gh`** REST from [`internal/gh`](../../internal/gh/). |

See also [Architecture review](architecture-review.md) for the full refactor roadmap.
